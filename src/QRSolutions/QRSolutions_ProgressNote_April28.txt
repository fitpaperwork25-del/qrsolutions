CURRENT BLOCKER (where we stopped)
------------------------------------
ISSUE: Webhook fires 200 OK but businesses table status never
       updates from "pending" to "active"

ROOT CAUSE IDENTIFIED:
The Stripe event carries:
  customer_email = fitpaperwork25@gmail.com
  metadata.userId = dd3eb013-81f1-4ae7-ac05-279f015f5ad6

But the test row in the businesses table has:
  email = etbarekh@me.com
  id = a57df375-95be-4273-859c-1e82b3737573

They don't match — so the update runs successfully but hits zero rows.

WEBHOOK CODE IS CORRECT. Data mismatch is the only problem.

THE FIX (one step, takes 2 minutes):
  1. Supabase → Table Editor → businesses table
  2. Find row with email = etbarekh@me.com
  3. Edit email → change to fitpaperwork25@gmail.com
  4. Save
  5. Stripe → Webhooks → playful-breeze → Event deliveries → Resend
  6. Supabase → businesses → confirm status = active

This is a test data issue. In production, real customers register
and pay with the same email, so this mismatch will not occur.
